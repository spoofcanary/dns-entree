dns-entree Widget Demo

All three tiers: Domain Connect, API credentials, and copy-paste fallback.

Configuration

Tier 1

Domain Connect (auto-detect)

Opens the widget, which calls /v1/detect and /v1/dc/discover. If the domain's DNS host supports Domain Connect, the one-click consent flow is shown. Try with a Cloudflare-hosted domain.

Tier 2

API Credentials

When Domain Connect is not available but a provider is detected (e.g., Route 53), the widget shows a credential form. The customer pastes their API token, records are pushed, and the widget verifies propagation.

Tier 3

Copy-Paste Fallback

When the API is unreachable or the provider is unknown, the widget shows each record as a card with copy buttons. No backend calls needed -- records come from your array. Try with a bad API URL.

Domain Connect Return

After a DC redirect, the widget auto-opens in verify mode. Simulate this by saving state and reloading with de_state param.
